Step 3 of 8 · Test Site Navigation

Navigation is often the first barrier users with disabilities encounter. For each task below, try it on the website you're reviewing, then rate how easy it was and describe any specific barriers.

i

Helpful Information

Why This Matters

Good navigation is essential for all users, but critical for those with disabilities. Testing real navigation tasks reveals barriers automated tests miss.

Screen Reader Tip

Test navigation using headings (H), landmarks (R), links lists (K), and search (Ctrl+F). Check if elements are properly labeled.

Keyboard Tip

Check if you can see where the focus is as you tab through the page. Is the focus order logical? Can you access all features?

Important

Check if navigation is consistent across different pages. Inconsistent navigation is a common barrier.

Task 1: Find the main product / service area

What to do: Locate the key features, products, or services offered.

Why this matters: Main content must be findable by everyone. For screen reader users, poor structure can hide important information. For mobility-impaired users, excessive tabbing creates fatigue. For users with cognitive disabilities, complex navigation can be confusing.

Tip: Try using both mouse and keyboard (Tab key) navigation. Screen reader users can try using headings list (H key) or landmarks.

Task 2: Locate contact details

What to do: Find phone numbers, email addresses, contact forms, or office locations.

Why this matters: Contact details must be accessible to everyone. For deaf users, non-phone options are essential. For users with cognitive disabilities, clear contact options reduce anxiety. For blind users, contact forms must work with screen readers.

Tip: Check the header, footer, and "Contact" pages. Screen reader users can search for terms like "contact" or "email".

Task 3: Find help or support

What to do: Identify where FAQs, help documentation, or customer support options are located.

Why this matters: Help resources are especially important for users with disabilities. Visually Impaired users may need to learn keyboard shortcuts. Users with cognitive disabilities may need simplified instructions. Users with motor limitations may need alternative ways to complete tasks.

Tip: Look for "Help," "Support," or "FAQs" in the main navigation or footer. Search functionality can also help locate these resources.

Task 4: Return to the home page

What to do: Test different ways to get back to the homepage (e.g., logo, home link, breadcrumbs).

Why this matters: Getting back to the homepage is essential when users get lost. Users with memory or attention difficulties need consistent navigation patterns. Screen reader users need properly labeled home links. Keyboard users need focusable navigation elements.

Tip: Try the website logo (is it keyboard accessible?), explicit "Home" links, and breadcrumb navigation if available. Screen reader users can check if the logo is properly labeled.
