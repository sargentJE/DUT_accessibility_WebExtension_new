Step 6 of 8 · Help & Support

Help and support resources must work for everyone. This step checks if you can get assistance when you need it. Try each task, then rate how easy it was and describe any barriers you found.

Helpful Information

Why This Matters

When users encounter problems, accessible help resources become critical. Without them, people with disabilities may be completely blocked.

Consider Your Needs

Think about your specific disability: Can you access help using your preferred tools and methods? Is support available in formats that work for you?

Testing Tip

Check if help documentation itself is accessible. Can you navigate it with your assistive technology? Are there multiple ways to get support?

Task 1: Find help resources

What to do: Locate help content such as FAQs, knowledge base, or tutorials.

  • Note number of clicks from homepage.
  • Check main navigation and footers.
  • Search using keywords like "help" or "support."

Why this matters for different users:

  • Screen reader users: Need properly labeled help links
  • Keyboard-only users: Need focusable help options
  • Cognitive disabilities: Need clearly visible help options
  • Low vision: Need help links with adequate contrast

Many users with disabilities need detailed guidance on accessibility features or alternative workflows.

Tip: Screen-reader users can search headings (H) or landmarks (R). Help links should be reachable from any page. Check if there are dedicated accessibility help sections.

Task 2: Check contact options

What to do: Evaluate support channels (email, phone, chat, ticket system).

  • Check support hours and response times.
  • Note alternative contact methods.
  • Look for self-service options (FAQs, forums).

Why multiple contact options matter:

  • Deaf or hard of hearing: Need text-based options (email, chat)
  • Speech impairments: Need alternatives to phone support
  • Anxiety disorders: May prefer non-real-time communication
  • Vision impairments: Need accessible contact forms

Users with different disabilities need appropriate contact methods that work with their specific needs.

Task 3: Read help articles / FAQs

What to do: Review help articles or FAQs for clarity.

  • Choose 2–3 topics.
  • Check language simplicity and use of visuals.
  • Assess organization and scannability.

Why accessible documentation matters:

  • Vision impairments: Need screen reader compatible formats
  • Cognitive disabilities: Need clear instructions with visual aids
  • Learning disabilities: Need step-by-step guidance without jargon
  • Motor limitations: Need documentation navigable by keyboard

Clear structure, plain language, and proper heading hierarchy are essential.

Task 4: Test a support request

What to do: Test a support channel (submit a question or test request).

  • Evaluate accessibility of the process.
  • Note barriers to requesting help.
  • Check for automated responses.

Why this matters: Support request forms and processes must be accessible to all users. Many support channels inadvertently create barriers with inaccessible forms, time limits, or CAPTCHA challenges.
